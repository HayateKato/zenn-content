## Ubuntu VMの作成
Ubuntu VM自体は下記にあるコミュニティスクリプトで作成できる。

https://community-scripts.org/scripts?q=ubuntu

```bash
bash -c "$(curl -fsSL https://raw.githubusercontent.com/community-scripts/ProxmoxVE/main/vm/ubuntu2504-vm.sh)"
```

しかし、上記スクリプトでは細かい設定を変更できない。
今回は後ほどGPUのパススルーの設定をしたいため、スクリプトは使わずに手動でVMを作成する。
まず、以下のURL先のUbuntu 26.04 LTSのisoファイルをダウンロードする。

https://ubuntu.com/download/server

local(<Proxmoxホスト名>)→isoイメージ→アップロードでダウンロードしたisoファイルをアップロードする。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.11.14.png)

成功すると以下のようなログが出てくる。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.16.19.png)

ubuntu-26.04のisoファイルが追加されていることを確認する。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.18.14.png)

次に、実際にVMを作成する。
Proxmoxホスト → VMを作成を選択し、各項目を以下のように設定する。 
設定には以下の記事を参考にした。

https://qiita.com/boxfish_jp/items/ea415296c2a79ef5ea0a

**全般**

- vm id：101(他のvmと被らないようにすればなんでもOK)
- 名前：ubuntu-vm(任意)

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.19.41.png)

**OS**

- ISO イメージ：アップロードしたubuntu26.04のisoイメージ

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.20.46.png)

**システム**

- BIOS/UEFI：OVMF (UEFI)

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.23.38.png)

**ディスク**

- パス/デバイス：SATA
- ディスクサイズ：40(任意)

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.25.43.png)

**CPU**

Proxmoxホストのcpuコア数が8なため、Proxmoxホスト：２、各Ubuntu VM：2、 OpenMediaVault VM：2という分配で考えている。

- コア：2
- 種別：host

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.28.31.png)

**メモリ**

Proxmoxホストに60GBあるため、余裕がある。
今回は各Ubuntu vmに8GB=8192MB割り当てることにする。
- メモリ (MiB)：8192
- Ballooningデバイス：チェックを外す

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.31.56.png)

**ネットワーク**

以前作成したVLAN(natnetからvnet1に変わっている)を使用する。
- ブリッジ：vnet1

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.32.58.png)

以下のようになっていることを確認する。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.34.37.png)

## Ubuntu の初期設定

作成したUbuntu VM → コンソールで初期設定を行う。
基本的にそのままでOK。
悩みそうなところだけ説明していく。

インストールするOSタイプはUbuntu serverを選択する。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.47.59.png)

ネットワークの設定はManualを選択して手動で設定する。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.49.26.png)

以下のように設定した。

- Subnet：10.10.10.0/24
- Address：10.10.10.v
- Gateway：10.10.10.1
- Name servers：空欄
- Search domains：空欄

今回プロキシは使わないので空欄でOK。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.53.53.png)

ストレージ設定はそのまま。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.54.57.png)

Ubuntu Proは使わないのでスキップする。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.58.38.png)

Proxmoxホストや手元PCからSSHでアクセスしたいので、Openssh serverをインストールする。

なお、鍵の設定はここでは行わない。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.59.09.png)

最後にRebootする。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_12.01.34.png)

再起動後、設定したユーザーとパスワードでログインできることを確認する。

次にネットワークの設定を行う。
/etc/netplan/*.ymlを編集して適切なDNSを追加する。

DNSは通常であればGoogleが提供している `8.8.8.8` などで大丈夫。

```diff
# Ubuntu VM上
~$ sudo cat /etc/netplan/00-installer-config.yaml
# This is the network config written by 'subiquity'
network:
  ethernets:
    ens18:
      addresses:
      - 10.10.10.v/24
      match:
        macaddress: bc:24:11:a1:43:22
      nameservers:
-       addresses: [8.8.8.8]
+       addresses: [<プライマリDNSのIP>, <セカンダリDNSのIP>]
        search: []
      routes:
      - to: default
        via: 10.10.10.1
      set-name: ens18
  version: 2
```

設定を反映、確認する。

```bash
# Ubuntu VM上
~$ sudo netplan apply
~$ resolvectl status
Global
         Protocols: -LLMNR -mDNS -DNSOverTLS DNSSEC=no/unsupported
  resolv.conf mode: stub

Link 2 (ens18)
    Current Scopes: DNS
         Protocols: +DefaultRoute -LLMNR -mDNS -DNSOverTLS DNSSEC=no/unsupported
Current DNS Server: <プライマリDNSのIP>
       DNS Servers: <プライマリDNSのIP> <セカンダリDNSのIP>
     Default Route: yes
```

名前解決できることを確認する。
