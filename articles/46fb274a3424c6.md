## プライベートネットワークの作成

Ubuntu VMを作る前に、VMにIPアドレスを割り振るため、プライベートネットワークを作成する。
仮想ルーターを作ってVLANを作り、プライベートIPを使用できるようにする。
以下の記事を参考にVLANを作る。

https://internet.watch.impress.co.jp/docs/column/shimizu/1585271.html

今回はDHCPやEVPNの設定はせず、とりあえず「**STEP 1-4：設定を適用する**」までと「**STEP 1-6：仮想マシンのネットワークで接続する**」のみ設定した。
DHCPの設定をしないため、「**[DHCPレンジ］タブで、仮想マシンに配布するIPアドレスを次のように指定する。**」は設定していない。

## Ubuntu VMの作成
Ubuntu VM自体は下記にあるコミュニティスクリプトで作成できる。

https://community-scripts.org/scripts?q=ubuntu

```bash
bash -c "$(curl -fsSL https://raw.githubusercontent.com/community-scripts/ProxmoxVE/main/vm/ubuntu2504-vm.sh)"
```

しかし、上記スクリプトでは細かい設定を変更できない。
今回は後ほどGPUのパススルーの設定をしたいため、スクリプトは使わずに手動でVMを作成する。
まず、以下のURL先のUbuntu 26.04 LTSのisoファイルをダウンロードする。

https://ubuntu.com/download/server

local(<Proxmoxホスト名>)→isoイメージ→アップロードでダウンロードしたisoファイルをアップロードする。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.11.14.png)

成功すると以下のようなログが出てくる。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.16.19.png)

ubuntu-26.04のisoファイルが追加されていることを確認する。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.18.14.png)

次に、実際にVMを作成する。

### Workerノード
まず、GPUクラスタ上の計算機に当たるWorkerノードとなるUbuntu VMを作成する。
Proxmoxホスト → VMを作成を選択し、各項目を以下のように設定する。
設定には以下の記事を参考にした。

https://qiita.com/boxfish_jp/items/ea415296c2a79ef5ea0a

**全般**

- vm id：100(他のvmと被らないようにすればなんでもOK)
- 名前：ubuntu-vm-worker(任意)

![](/images/46fb274a3424c6/スクリーンショット_2026-05-12_20.14.22.png)

**OS**

- ISO イメージ：アップロードしたubuntu26.04のisoイメージ

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.20.46.png)

**システム**

- BIOS/UEFI：OVMF (UEFI)
- Pre-Enroll keys：チェックを外す

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.23.38.png)

**ディスク**

- ディスクサイズ：40(任意)

![](/images/46fb274a3424c6/スクリーンショット_2026-05-12_20.15.01.png)

**CPU**

Proxmoxホストのcpuコア数が8なため、Proxmoxホスト：２、各Ubuntu VM：1、OpenMediaVault VM：1という分配で考えている。

- コア：1
- 種別：host

![](/images/46fb274a3424c6/スクリーンショット_2026-05-12_20.15.23.png)

**メモリ**

Proxmoxホストに60GBあるため、余裕がある。
今回は各Ubuntu vmに8GB=8192MB割り当てることにする。
- メモリ (MiB)：8192
- Ballooningデバイス：チェックを外す

Ballooningデバイス -> オンにするとProxmoxホストがVMのメモリを動的に増減させる。VMから見るとメモリの量が勝手に変動することになり、VMの動作が不安定になる可能性がある。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.31.56.png)

**ネットワーク**

作成したVLANを使用する。
- ブリッジ：vnet1

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.32.58.png)

以下のようになっていることを確認する。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-12_20.17.37.png)

## Ubuntu の初期設定

作成したUbuntu VM → コンソールで初期設定を行う。
基本的にそのままでOK。
悩みそうなところだけ説明していく。

インストールするOSタイプはUbuntu serverを選択する。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.47.59.png)

ネットワークの設定はens18 eth -> Edit IPv4 -> Manualを選択して手動で設定する。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-12_19.37.57.png)

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.49.26.png)

以下のように設定した。

- Subnet：10.10.10.0/24
- Address：10.10.10.v
- Gateway：10.10.10.1
- Name servers：使用可能なDNSサーバーのIP(Googleが提供している8.8.8.8など)
- Search domains：空欄

今回プロキシは使わないので空欄でOK。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.53.53.png)

ネットワーク設定が適切に行われていると、パッケージ情報を取得できている。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-12_19.43.11.png)

ストレージ設定はそのまま。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.54.57.png)

確認画面が出てくるがそのまま進む。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-12_20.25.40.png)

Your name と Pick a usernameは同じもので大丈夫。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-12_20.28.11.png)

Ubuntu Proは使わないのでスキップする。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.58.38.png)

Proxmoxホストや手元PCからSSHでアクセスしたいので、Openssh serverをインストールする。
なお、鍵の設定はここでは行わない。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_11.59.09.png)

インストール推奨のパッケージ一覧が出てくる。
何もいじらずに進む。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-12_19.48.15.png)

最後にRebootする。

![](/images/46fb274a3424c6/スクリーンショット_2026-05-03_12.01.34.png)

再起動後、設定したユーザーとパスワードでログインできることを確認する。
アップデートしておく。
```bash
# Ubuntu VM上
~$ sudo apt update
~$ sudo apt upgrade
```
